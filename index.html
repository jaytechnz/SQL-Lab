CREATE DATABASE 
CREATE DATABASE database_name;

Creates a new database. Use a descriptive name.

CREATE TABLE 
CREATE TABLE TableName (
  col1  INTEGER PRIMARY KEY,
  col2  VARCHAR(50),
  col3  REAL,
  col4  BOOLEAN,
  col5  DATE,
  col6  TIME,
  col7  CHARACTER,
  FOREIGN KEY (col2) REFERENCES Other(col2)
);

Data types:

INTEGERWhole numbers, e.g. 42
REALDecimals, e.g. 3.14
VARCHAR(n)Variable text, max n chars
CHARACTERSingle character, e.g. 'A'
BOOLEANTRUE or FALSE
DATEe.g. '2026-04-22'
TIMEe.g. '09:30:00'
ALTER TABLE 
-- Add a column:
ALTER TABLE TableName
  ADD column_name TYPE;

-- Example:
ALTER TABLE Students
  ADD email VARCHAR(100);

Used to change an existing table definition without recreating it.

PRIMARY KEY

Inline syntax:

id INTEGER PRIMARY KEY

Constraint syntax (end of column list):

PRIMARY KEY (id)

Uniquely identifies each row. Cannot be NULL. Each table has at most one primary key.

FOREIGN KEY 
FOREIGN KEY (field)
  REFERENCES Table (Field)

Example:

CREATE TABLE Loans (
  loan_id   INTEGER PRIMARY KEY,
  member_id INTEGER,
  FOREIGN KEY (member_id)
    REFERENCES Members(member_id)
);

Links a column in one table to the primary key in another. The referenced table must exist first.

SELECT 
-- All columns:
SELECT * FROM table;

-- Specific columns:
SELECT col1, col2 FROM table;

-- With alias:
SELECT col1 AS alias FROM table;

-- With WHERE:
SELECT * FROM table
WHERE col = 'value';

-- Multiple conditions:
SELECT * FROM table
WHERE col1 = 'x' AND col2 > 10;

SELECT * FROM table
WHERE col1 = 'x' OR col1 = 'y';

-- ORDER BY:
SELECT * FROM table
ORDER BY col ASC;   -- or DESC

-- INNER JOIN:
SELECT a.col, b.col
FROM tableA AS a
INNER JOIN tableB AS b
  ON a.id = b.a_id;

-- GROUP BY:
SELECT col, COUNT(*) AS n
FROM table
GROUP BY col;
Aggregate Functions
COUNT(*)Count all rows
COUNT(col)Count non-NULL values
SUM(col)Total of numeric column
AVG(col)Mean of numeric column
MAX(col)Largest value
MIN(col)Smallest value

Example:

SELECT department,
  COUNT(*) AS num_staff,
  AVG(salary) AS avg_sal
FROM employees
GROUP BY department
ORDER BY avg_sal DESC;
INSERT INTO 
-- Specify columns:
INSERT INTO Table (col1, col2, col3)
VALUES (val1, 'text', 3.14);

-- All columns (order matters):
INSERT INTO Table
VALUES (1, 'Alice', 25);

String values use single quotes. Numeric values do not.

UPDATE 
UPDATE Table
SET col1 = new_value,
    col2 = col2 * 1.1
WHERE condition;

Always use WHERE — without it, every row is updated.

DELETE FROM 
DELETE FROM Table
WHERE condition;

Always use WHERE — without it, every row is deleted.

NULL Values 
-- Check for NULL:
WHERE col IS NULL
WHERE col IS NOT NULL

-- NULL ≠ empty string ≠ 0

NULL means "unknown" or "missing". You cannot use = NULL — always use IS NULL.

Quick Syntax Checklist
  • String values use single quotes 'text'
  • Create parent tables before child tables
  • FOREIGN KEY references must match the parent's PRIMARY KEY type
  • GROUP BY must include all non-aggregated SELECT columns
  • Use IS NULL / IS NOT NULL (never = NULL)
  • ORDER BY comes after GROUP BY
  • Always use WHERE with UPDATE and DELETE
